# Sub-issue 15: `Interest` builder accepts invalid date combinations silently

**Severity:** Low / Style

**File:** `src/accounting/interest.rs`

## Description

The `year()`, `month()`, `day()` builder methods accept arbitrary `u32` values. An impossible
date like (February 30) will produce a `None` only deep inside the calculation, making
debugging difficult.

## Suggestion

Validate the day against the month (and year for leap-year aware checks) in `day()` and
return `Option<Self>` (or `Result<Self, ...>`) on invalid input. This surfaces the error at
the point of construction rather than deep in the calculation.
