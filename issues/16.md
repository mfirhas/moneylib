# Sub-issue 16: `calendar.rs` — `current_date()` depends on the system clock

**Severity:** Low / Style

**File:** `src/calendar.rs`

## Description

The default start date for interest calculations is taken from `SystemTime::now()`.
Tests that use the default date are non-deterministic and can produce different results on
different days (e.g. leap-year boundaries, month-length differences).

## Suggestion

- Accept an explicit date parameter and use the "current date" only as a convenience default,
  **or**
- Make the clock injectable (e.g. via a trait or a function pointer) so tests can supply a
  fixed date.
