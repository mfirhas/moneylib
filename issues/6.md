# Sub-issue 6: `ExchangeRates::set()` accepts unvalidated `&str` currency codes

**Severity:** High

**File:** `src/exchange.rs` (line 312)

## Description

Arbitrary strings (e.g. `"USE"` instead of `"USD"`) are accepted without validation.
Later `convert` / `get_pair` calls will simply return `None`/`Err` with no clue about
the root cause.

## Suggestion

- Validate that the code is a known ISO 4217 alphabetic code (e.g. against `currencylib`), **or**
- Accept a type-safe `C: Currency` parameter and use `C::CODE` instead of `&str`.
