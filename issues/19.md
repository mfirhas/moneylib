# Sub-issue 19: `BaseMoney::is_positive()` documentation inconsistency

**Severity:** Low / Style

**File:** `src/base.rs` (line 337)

## Description

The doc comment for `BaseMoney::is_positive()` says "Returns `true` if the amount is positive"
and the example only shows non-zero cases. Meanwhile, `ObjMoney::is_positive()` (line 103 of
`src/obj_money/mod.rs`) correctly documents "(or zero)".

These two definitions are inconsistent: one implies zero is not positive, the other explicitly
states it is.

## Suggestion

Align the documentation of `BaseMoney::is_positive` with the actual behavior:
either state explicitly that zero is considered positive (matching `ObjMoney`), or fix the
implementation to return `false` for zero (see sub-issue #2).
