# Sub-issue 3: `Money * Money` and `Money / Money` have incorrect financial semantics

**Severity:** Critical / Bug

**Files:**
- `src/ops.rs` (lines 45–79)
- `src/raw_money/ops.rs` (lines 45–79)

## Description

`Money * Money` produces a "dollars-squared" result that has no financial meaning.
`Money / Money` should yield a dimensionless `Decimal` ratio, not another `Money`.
These operators are still present and will silently produce nonsensical results.

## Suggestion

- Remove `Mul` and `Div` for money×money.
- If a dimensionless ratio is needed, provide a dedicated method returning `Decimal`.
- Note: the `BaseOps` trait super-bounds force these operators to exist (see sub-issue #4),
  so fixing this requires relaxing `BaseOps` as well.
