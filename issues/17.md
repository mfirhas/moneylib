# Sub-issue 17: Missing `#[non_exhaustive]` on `MoneyError`

**Severity:** Low / Style

**File:** `src/error.rs`

## Description

Adding a new error variant in a future version will be a breaking change for downstream users
who exhaustively `match` on `MoneyError`. Without `#[non_exhaustive]`, any future variant
addition is a semver-breaking change.

## Suggestion

Add `#[non_exhaustive]` to the `MoneyError` enum so that downstream users are required to
include a wildcard arm in their match expressions, allowing new variants to be added without
breaking semver compatibility.
