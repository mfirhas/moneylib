# Sub-issue 18: `clone()` called on `Copy` types

**Severity:** Low / Style

**Files:**
- `src/percent_ops.rs` (lines 219, 221, 240, 241)
- `src/iter_ops.rs` (various)
- `src/money.rs` (line 496)

## Description

`Money` and `RawMoney` derive `Copy`, so calling `.clone()` on them is a no-op copy.
While harmless, it adds noise and implies the type might not be `Copy`.

## Suggestion

Replace `.clone()` calls on `Money` / `RawMoney` values with direct use or a simple
`let` binding, relying on the `Copy` semantics instead.
